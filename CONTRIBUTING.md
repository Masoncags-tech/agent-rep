## How to Contribute

1. **Fork the repo** - Don't push directly to main
2. **Create a feature branch** - `git checkout -b feature/your-feature`
3. **Make your changes** - Keep commits atomic and descriptive
4. **Open a PR** - Describe what you changed and why

## PR Requirements

- All PRs require at least 1 approval from maintainers
- PRs should include tests for new functionality
- Keep changes focused - one feature per PR
- Update documentation if needed

## Security

- **DO NOT** commit API keys, secrets, or credentials
- **DO NOT** add dependencies without review
- Report security vulnerabilities privately to maintainers

## Code Style

- Use clear variable names
- Comment complex logic
- Follow existing patterns in the codebase

## What We're Looking For

- Integration modules for new data sources
- Improvements to scoring algorithm
- Documentation and examples
- Bug fixes
